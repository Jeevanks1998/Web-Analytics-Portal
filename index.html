Privacy-First Analytics

All analytics implementations must comply with GDPR, CCPA, and applicable data protection regulations. This section provides guidance on handling Personally Identifiable Information (PII) within our analytics stack.

🔑

What is PII in GA4?

Personally Identifiable Information includes any data that can directly or indirectly identify an individual — names, email addresses, phone numbers, IP addresses, user IDs linked to real people, and device identifiers.

  • Full names and email addresses
  • Precise geographic locations
  • Device identifiers & IP addresses
  • Login credentials or user IDs
  • Financial or health information
Google's PII Policy →
⚠️

PII Prohibitions in GA4

Google Analytics prohibits sending PII to GA4 as it violates their Terms of Service. Violations can result in account termination and regulatory penalties.

  • Never send email addresses in event parameters
  • Do not pass raw user IDs that link to PII
  • Avoid collecting form field values directly
  • Redact PII before sending to the datalayer
  • Review URL parameters for embedded PII
GA4 Terms of Service →
🛠️

Technical Controls & GTM

Use GTM variables and custom JavaScript to strip or hash PII before it reaches GA4. Implement server-side tagging for additional data control.

  • Use hashed user IDs (SHA-256)
  • Configure URL redaction in GA4 settings
  • Enable IP anonymisation
  • Set data retention to minimum required
  • Use GTM PII detection variables
GA4 Config Reference →
📋

Consent & Legal Basis

Analytics tracking must have a valid legal basis. Consent Mode v2 integrates with OneTrust and Commanders Act to respect user consent choices.

  • Implement Consent Mode v2 for GA4
  • Honour opt-out signals (GPC)
  • Maintain consent audit logs
  • Update privacy notices accordingly
  • Conduct regular DPIAs for new tracking
Consent Mode Documentation →
📁

Data Subject Rights

Users have the right to access, correct, and delete their data. GA4 provides mechanisms to support DSARs (Data Subject Access Requests).

  • Use GA4 User Deletion API
  • Document all data collection points
  • Respond to DSARs within 30 days
  • Maintain data processing records
  • Appoint a DPO for large-scale processing
User Deletion API →

Audit & Compliance Checklist

Regular audits ensure ongoing compliance. Use this checklist to review your GA4 implementation for PII risks.

  • ✅ DebugView review for PII leaks
  • ✅ URL parameter audit quarterly
  • ✅ Custom dimension/metric review
  • ✅ Third-party integrations assessed
  • ✅ Consent banner A/B test compliance
